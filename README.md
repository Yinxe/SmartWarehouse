# SmartWarehouse

Minecraft Bedrock Edition 智能仓库管理 Addon。基于 Script API 实现自动分拣、容器整理、仓库统计等功能。

> **版本**: 0.0.55 | **许可证**: MIT
> **项目地址**: https://github.com/YinxSmartHouse/SmartWarehouse

> **测试版本** — 可能存在未发现的 bug，使用前请备份世界。

---

## 功能一览

### 仓库管理
- **创建仓库** — 交互式选区，手持信物对空右键打开主菜单，或 `/sw:create` 命令划定区域
- **调整区域** — 设置页随时修改仓库边界，支持缩放
- **删除仓库** — 设置页一键删除（不可撤销）
- **边界光幕** — 开启后显示仓库 12 条棱的粒子线框，便于定位
- **多仓库支持** — 不同维度、不同区域可同时管理多个独立仓库，支持多玩家
- **合箱/拆箱** — 大箱子合并/拆分为单箱，自动继承角色/启用状态/发现时间
- **方块事件维护** — 放置/破坏容器方块自动增量更新仓库数据

### 自动分拣系统
- **五级优先级路由**：大宗容器 -> 同类容器 -> 同族容器 -> 自动创建分类 -> 杂项兜底
- **家族白名单** — 仓库设置中勾选需要自动分拣的 51 个物品家族
- **轮询调度** — 多个输入容器间负载均衡，每 interval 处理一格
- **区块加载预检** — 区块未加载时静默跳过，下次调度周期重试（取决于处理速度）
- **索引自愈** — 自动维护物品类型索引，脏索引零回合自动修复
- **惰性生命周期** — 玩家接近时激活仓库，无人时自动休眠，脚本启动零开销
- **全局分拣总开关** — 管理员可暂停/恢复全部分拣

### 容器整理
- **SlotOrganizer** — 三段式 API（分析 -> 应用 -> 一步到位）
- **智能混乱度评分** — 相邻逆序对（70%）+ 堆叠优化（30%）双重评估
- **自动整理** — 分拣后自动触发，阈值 0~100 可调（0=每次整理 ~ 100=永不）
- **即时整理** — 容器设置面板中的"立即整理"按钮
- **快速整理** — 潜行 + 信物 + 右键容器整理该容器；潜行+信物+右键非容器方块整理背包
- **背包整理** — `/sw:organize` 一键整理玩家背包（快捷栏除外）
- **写锁保护** — 容器级写锁（100 tick 超时），防止整理与分拣冲突

### 容量预警（三级预警系统）
- **黄色预警** — 单容器容量 >= 90%，提示具体容器
- **红色预警** — 某类容器组全部满仓，物品被降级
- **深红预警** — 所有容器全满，物品无法分拣
- 冷却 100 tick（~5秒）防刷屏，容器级独立开关

### 仓库统计
- **容器概况** — 设置页顶部显示各角色容器数量与容量概览
- **容器详情** — 信物点击容器查看方块类型、槽位、物品数、种类、混乱度、家族纯度
- **统计缓存** — 写穿透（内存 + DynamicProperty），仅在被显式失效时重算

### 物品分类
- **51 个家族** — 全部 1430+ 可分类物品按用途/来源划入互斥家族
- **家族纯度排序** — 同族物品容器按纯度降序排列
- 完整分类列表见 [docs/item-family-guide.md](docs/item-family-guide.md)

### 容器搜索
- **中文名/英文名/物品ID 模糊搜索** — 在主菜单中打开搜索界面
- **粒子位置标记** — 匹配容器自动播放紫色粒子标记（10 秒，手持信物续时）

### 管理功能
- **模组配置** — 信物切换、仓库体积上限、容器数上限、全局速度限制
- **全服统计** — 查看所有玩家的仓库和容器数量排行

## 容器角色

| 角色 | 标签 | 说明 |
|------|------|------|
| **input** | 输入 | 玩家存入待分拣物品，分拣引擎从此取物。多 input 轮询调度 |
| **normal** | 普通仓位 | 主存储，按物品类型自动归类存放已有同类物品 |
| **bulk** | 大宗仓位 | 单物品专用，按箱内首个物品类型匹配。空箱需手动放物设定 |
| **misc** | 其他仓位 | 杂项容器，无法归类到普通容器的物品最终流向此处（兜底） |

漏斗自动识别为 **input** 角色且不可更改。容器角色和启用状态在 UI 中实时切换。

## 命令

| 命令 | 描述 | 权限 |
|------|------|------|
| `/sw:create <名称> <x1> <y1> <z1> <x2> <y2> <z2>` | 创建仓库 | op |
| `/sw:resize <名称> <x1> <y1> <z1> <x2> <y2> <z2>` | 调整仓库区域 | op |
| `/sw:rescan <名称>` | 重新扫描仓库容器 | op |
| `/sw:rescan_preview <名称>` | 预览重扫变更而不实际修改 | op |
| `/sw:delete <名称>` | 删除仓库 | op |
| `/sw:organize` | 整理玩家背包 | 所有人 |
| `/sw:menu` | 打开 SmartWarehouse 主菜单 | 所有人 |
| `/sw:search <关键词>` | 在附近且属于该玩家的仓库中搜索物品 | op |

## 界面操作

### 主菜单（/sw:menu 或手持信物对空右键）
- **容器搜索** — 打开搜索界面输入关键字
- **管理仓库** — 智能定位附近且属于该玩家的仓库，直达设置页
- **仓库列表** — 列出所有仓库（管理员看全部，非管理员只看自己的）
- **创建仓库** — 启动创建流程（需 op）
- **设置** — 模组配置面板（仅管理员）

### 容器设置菜单（手持信物右键容器）
- **非管理员** — 只读查看容器信息
- **管理员** — 可修改：启用/禁用、角色切换（漏斗不可改）、容量预警开关、立即整理

### 仓库设置页面
- **基础设置**：仓库名称、默认新容器角色、默认启用状态
- **分拣设置**：处理速度、自动创建分类、启用/禁用仓库
- **显示设置**：边界光幕、自动整理阈值、容量预警
- **操作**：刷新容器、修复仓库、删除仓库、刷新统计、家庭成员配置、调整区域

## 快速开始

1. 将行为包和资源包导入 Minecraft
2. 在世界设置中启用行为包和资源包
3. 给自己添加 op 标签：
   ```
   /tag @s add op
   ```
4. 手持信物（默认木锄）对空右键打开主菜单
5. 创建仓库 -> 选择两个对角点 -> 完成创建
6. 设置一个 input 容器，放入物品开始自动分拣

### 权限说明

| 操作 | 权限要求 |
|------|---------|
| 管理命令（create/resize/rescan/delete/search） | op 标签 |
| 公共命令（organize/menu） | 所有人 |
| 信物交互容器菜单 | 非管理员只读，管理员可修改 |
| 仓库设置修改 | 必须是仓库所有者或管理员 |

## 构建与开发

### 环境要求
- Node.js >= 18
- npm >= 9

### 构建步骤

```bash
npm install
npx tsc
```

编译产物输出到 `lib/` 目录，需手动部署到行为包的 `scripts/` 目录。

### 一键打包（.mcaddon）

```bash
npm run mcaddon
```

输出文件位于 `dist/packages/`。

### 代码生成工具

| 工具 | 说明 |
|------|------|
| `tools/generateItemFamilies.mjs` | 重新生成物品家族分类 |
| `tools/annotateFamilies.mjs` | 为 ItemFamilies.ts 注入中文注释 |
| `tools/generateNameMap.mjs` | 生成中文名映射表 |
| `tools/statsFamilies.mjs` | 统计家族覆盖情况 |
| `tools/findDups.mjs` | 查找物品 ID 重复 |
| `tools/findUncategorized.mjs` | 查找未分类物品 |

修改流程：编辑生成器 -> `node generateItemFamilies.mjs` -> `node annotateFamilies.mjs` -> `npx tsc`

## 项目结构

```
BP/                   行为包（manifest.json + scripts 编译产物）
RP/                   资源包（manifest.json + 纹理/声音）
scripts/              TypeScript 源码
  main.ts             入口文件（初始化依赖、注册事件和命令）
  types.ts            集中式类型定义（40+ 类型/接口）
  version.ts          自动生成的版本号与构建时间
  commands/           命令路由层（8 条自定义命令）
  data/               数据文件（51 家族物品分类、中文名映射）
  interaction/        工具交互（信物右键、方块事件处理）
  organize/           容器整理器（混乱度评分、三段式 API）
  runtime/            运行时缓存层（内存索引、惰性重建）
  sorting/            分拣引擎和调度器（含容量预警、事务日志）
  storage/            持久化层（DynamicProperty 世代分片存储）
  ui/                 玩家交互界面（11 个 UI 模块）
  util/               工具函数（日志、坐标、JSON、权限、钩子）
  warehouse/          核心业务逻辑（仓库 CRUD、容器扫描、搜索）
tools/                维护工具（6 个 Node.js 脚本）
docs/                 文档（架构概览、分类指南、路由流程说明）
  api/                Minecraft Script API 类型声明
  superpowers/        设计提案与规格文档
tests/safety/         安全测试（回滚、快照、整理器）
```

## 技术栈

- **语言**：TypeScript（ES6 target, strict 模式）
- **运行时**：Minecraft Bedrock Script API（@minecraft/server ^2.6.0）
- **UI**：@minecraft/server-ui（ActionForm / ModalForm）
- **构建**：TypeScript 编译器 + just-scripts
- **代码规范**：Prettier + ESLint（eslint-plugin-minecraft-linting）
- **测试**：自建 Node.js 测试运行器
- **文档**：docs/ 目录（架构概览、路由流程、分类指南、API 类型声明、设计提案）
